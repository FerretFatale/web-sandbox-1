Developer Quick Start

Set up The Vault runtime, run the test suite, and make your first tool call in minutes.

Get started → View on GitHub → 
# 1. Install dependencies
pip install -r requirements.txt

# 2. Run the full test suite
python -m pytest Copilot_Tests/ -q

# 3. Boot the vault dashboard
python vault_dashboard.py
from vault_brain import api_read_mission_state
from custom_tools_registry import get_all_tools

# Check active mission
mission = api_read_mission_state()
print(mission["title"], mission["status"])

# List all registered tools
tools = get_all_tools()
print(f"{len(tools)} tools registered")

Requires Python 3.11+ and a local .env file with API keys. See requirements.txt for the full dependency list. Runtime Churn is controlled by deterministic tests, explicit approval gates, and mission-state checkpoints.

Personas

View all 20 →

The Vault routes work to 20 purpose-built personas via persona_manifest.json. Each persona carries a core_skill, explicit tool grants, and delegation rules. TaskMaster_Brain uses core_skill for routing — a missing or vague value causes the planner to treat all personas as identical.

Orchestration 21 tools
Vault_Brain

System coordinator. Mission-state authority, persona routing, Gatekeeper governance, and deploy guard (SR-002). Temperature 0.0. Does not execute domain work.

Security guide →

Personas may only delegate via explicit allowed_delegations in the manifest. No persona may grant tools not in custom_tools_registry.py. Deploy tools are system_only — Vault_Brain exclusive. Tool risk classes: read_only_local, local_write, external_write, deploy, approval_gated, system_only — defined in docs/manual §G.

Core components

View architecture →

Start with vault_brain.py for orchestration, or TaskMaster_Brain.py for planning-first workflows. All components are Pylance-validated and have deterministic test coverage, with policy/behavior constraints sourced from Contextual_Thinking.

Orchestration
vault_brain.py

Mission state authority, persona routing, and Gatekeeper governance. The runtime anchor for all task decisions, approval gates, and tool-call dispatch.

 Planning
TaskMaster_Brain.py

Decomposes tasks into ordered steps, controls sequencing, manages checkpoint state, and drives the execution loop from first-plan through to verified completion.

 Consensus
Consensus_Engine/

Multi-model governance scoring. Runs policy checks before any promoted action, publish decision, or external write. Produces structured verdicts: PASS, PASS_WITH_NOTES, or FAIL.

Start building

🔧
Register a tool →

Add a callable function to custom_tools_registry.py and expose it to all personas via the tool registry.
🧪
Write deterministic tests →

Add coverage to Copilot_Tests/ using the project's pytest fixtures, contract helpers, and 80+ existing test patterns.
🎭
Define a persona →

Add a persona to persona_manifest.json with tool grants, model assignments, and explicit scope rules.
🗺️
Work with mission state →

Read, write, and checkpoint .kaos_mission.json via state_io_manager.py for context-safe execution across sessions.

🔀
Extend routing rules →

Modify Toolkit/omni_router.py to add model/tool routing logic for new capability lanes. Routing changes require test coverage.
📖
Publish a manual update →

Edit source pages, run quality gates, and publish to web-sandbox-1 via manual_publisher.py with approval gating.

✅
Run Gatekeeper review →

Execute governance checks via the Consensus Engine before promoting sandbox work to production. Produces a dated, archived verdict artifact.
🔌
Wire an integration →

Add external integrations to Toolkit/ following the approved-first, dry_run=True default pattern.
