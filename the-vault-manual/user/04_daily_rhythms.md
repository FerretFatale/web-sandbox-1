# Daily Rhythms

## Daily cadence

1. Morning: review briefing, blockers, top 3 outcomes.
2. Midday: close one mission lane and one maintenance lane.
3. Evening: reconcile partials, queue next-day lanes.

## Daily operator checklist

1. Check active backlog state.
2. Check human-input blockers.
3. Execute highest-value lane first.
4. Capture decisions in canonical docs.
5. Run relevant deterministic tests before close.

## Human input flow

Use Human_Input surfaces when blocked by:
1. approvals,
2. credentials,
3. external account mutations,
4. unresolved authority ambiguity.

## Weekly cadence

1. governance hygiene and manual freshness,
2. broader test sweep,
3. stale-lane cleanup and prioritization.
