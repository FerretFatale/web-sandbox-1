# Workflow Playbooks

## Playbook 1: Daily Operations

1. Review priorities and blockers.
2. Select one primary outcome lane.
3. Execute with checkpoint criteria.
4. Validate, document, and close.

## Playbook 2: Proposal-First Change

1. Define objective and constraints.
2. Produce scoped proposal with risks.
3. Approve path (implement/defer/split).
4. Execute smallest safe tranche.

## Playbook 3: Manual Overhaul Lane

1. Lock references and target audience.
2. Build page map and rewrite backlog.
3. Implement source-first.
4. Validate split/publisher tests.
5. Prepare/publish/verify with approval.

## Playbook 4: Blocked Lane Handling

1. Record blocker and risk level.
2. Identify exact human action needed.
3. Complete all internal prework.
4. Resume immediately after input.

## Playbook 5: External Capability Gap

1. Capture gap and external surface used.
2. Execute minimal successful path.
3. Classify assimilation outcome: Adopt/Scaffold/Defer.
4. Enshrine governance update.
