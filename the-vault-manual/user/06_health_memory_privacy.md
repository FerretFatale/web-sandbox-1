# Health, Memory, and Privacy

## Privacy baseline

1. Local-first data handling is default.
2. Sensitive data should not be published.
3. External writes require explicit authorization lanes.

## Health and memory boundaries

1. Health tracking is operational support, not clinical diagnosis.
2. Memory supports continuity; always verify high-stakes claims.
3. Personal artifacts remain outside public docs unless explicitly sanitized.

## Safe sharing checklist

Before sharing output externally:
1. remove personal identifiers,
2. remove private operational logs,
3. remove credentials and secrets,
4. confirm public-safe scope.

## Trust model

The_Vault should explain what it can and cannot do, and surface blockers instead of guessing.
