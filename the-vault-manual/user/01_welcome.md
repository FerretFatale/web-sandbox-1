# Welcome - What is The_Vault?

The_Vault is a local-first personal AI operating system.

In plain language: it is a coordinated team of AI specialists that helps you turn messy ideas into ordered action, while keeping you in control of risky decisions.

## Why this manual exists

This manual teaches you how to operate The_Vault safely and effectively, even if you are new to AI systems.

You do **not** need advanced technical knowledge to use this guide.

## Core idea in one minute

The_Vault does four important things:

1. **Listens to your request** (idea, task, blocker, question).
2. **Routes work to the right specialist persona** instead of using one generic assistant.
3. **Executes in steps** (especially for bigger, multi-part work).
4. **Stops for approval** when actions could affect external systems or high-risk areas.

## What makes it different from a normal chatbot

1. It is not only for conversation - it supports structured execution.
2. It uses personas with defined roles and boundaries.
3. It treats safety and approval as first-class requirements.
4. It supports checkpoint-based progress and deterministic validation.
5. It is built around local project authority and explicit governance.

## What you can use it for (beginner examples)

1. **Daily planning**
	- "Help me choose my top 3 priorities today and break them into steps."
2. **Writing and documentation**
	- "Rewrite this section for a beginner audience and keep technical details accurate."
3. **Task execution support**
	- "Run this lane in two phases and stop after phase one for review."
4. **Risk-aware operations**
	- "Do a dry run first and list blockers before any live action."

## How control works

You are always the final decision-maker for risky or external actions.

If something needs approval, The_Vault should surface that clearly instead of silently doing it.

## Beginner success rule

When in doubt, ask for:

1. a short plan,
2. a dry run,
3. explicit risks,
4. and a stop point before live changes.
