Feature Showcase

The Vault in motion

These are not demos. They are real capabilities running on real data in a real operating system. Each spotlight pairs a signal, a mechanism, a boundary, and evidence.

How to read this page: each feature block shows what makes it impressive (Signal), how it actually works (Mechanism), what is intentionally constrained (Boundary), and where the proof is (Evidence). Display posture: demonstrating quality, not selling access.
01 / Architecture

Vault Brain Orchestration

Every request entering The Vault is routed through Vault Brain to the correct specialist domain. No domain bypasses the governance chain. No action is taken without a traceable decision path.

Signal
44 specialised capabilities routed through a single governed orchestration layer
Mechanism
vault_brain.py with omni_router, domain-specific tool registries, persona manifest
Boundary
External writes require explicit human approval; dry-run-first patterns enforced at code level
Evidence
KAOS_CONTRACTS.json, persona_manifest.json, Copilot_Tests/ suite (400+ deterministic tests)

02 / Capabilities

Health Intelligence

Oura Ring biometric data feeds directly into scheduling, task prioritisation, and routine adjustment. The system adapts to real physical state, not aspirational plans.

Signal
Wearable biometrics drive scheduling and cognitive load decisions in real time
Mechanism
Oura API integration, Health/ domain, readiness and sleep scoring fed to Vault Brain context
Boundary
Health data stays local; never sent to external services; no AI diagnosis claims
Evidence
Health/ folder, Routines/ integration, Oura connector code in Skills_Index.json
View in Feature Reference →
03 / Business + Creative

Living Business Plan

The business plan is not a static document. It updates from real revenue data, grant tracker activity, and business operations logs. The document reflects what is actually happening, not what was written six months ago.

Signal
Business plan self-updates from live operational data; always reflects current state
Mechanism
Business/living_business_plan.py, revenue_framework.py, grant_tracker.py, business_ops.py
Boundary
No external publishing without approval; financial projections are internal planning only
Evidence
Business/ folder, brand_registry.json, business_milestone_tracker.py

04 / Capabilities

Email Intelligence

Proton Bridge IMAP integration with pattern-based routing, contact allowlists, urgency classification, and hold-folder policy for uncertain clusters. Zero silent deletions, full auditability.

Signal
Governed email triage with urgency dimensions, spam-cluster scanning, and protected sender lists
Mechanism
Toolkit/email_intelligence.py, Toolkit/proton_bridge.py, contact_allowlist.json, IMAP local integration
Boundary
Proton Bridge only; no cloud email APIs; hold-folder policy prevents irreversible deletes
Evidence
Copilot_Tests/test_email_intelligence.py (457 tests passing), proton_bridge.py, hold-folder protocol
View in Feature Reference →
05 / Capabilities

Research Pipeline

Web search feeds synthesis, which feeds the Inbox Router, which routes findings into goals, schedules, and planning surfaces. Research does not dead-end in a folder.

Signal
Research automatically integrates into planning workflows rather than accumulating as passive notes
Mechanism
Websearch/ domain, Research/ domain, synthesis routing through Inbox Router, Goals/ integration
Boundary
Search is read-only; no automated posting or sharing; all outputs require human review before action
Evidence
Websearch/ folder, Research/ folder, Inbox Router wiring in vault_brain.py
View in Feature Reference →
06 / Architecture

Governance Engine

Gatekeeper Reviews, mission state tracking, KAOS_CONTRACTS.json, and KAOS_TEST_MATRIX.json form a governance layer that prevents rogue behaviour, off-mission execution, and unverified promotions from sandbox to production.

Signal
Every major operation is governed, traceable, and reversible before commit
Mechanism
.kaos_mission.json, KAOS_CONTRACTS.json, KAOS_TEST_MATRIX.json, Consensus_Engine/, gatekeeper_process_intelligence_log.md
Boundary
KAOS is sandbox-only; The_Vault is production-authoritative; promotion requires test-pass evidence
Evidence
docs/copilot-output/reference/30-gatekeeper-reviews/, docs/copilot-output/reference/40-intelligence-logs/, AGENTS.md
