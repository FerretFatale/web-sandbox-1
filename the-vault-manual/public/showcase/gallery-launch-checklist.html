Public Showcase · Launch Checklist

A launch checklist that keeps the showcase honest and usable

The showcase should only go live when it is navigable, readable, and grounded in real evidence. This page is the gate that prevents future-marker drift and placeholder surfaces from sneaking back in.

Content gate

Gate C

Real content only

Every card must point to an actual page and every page must add value beyond a header.

Gate C

Evidence-backed copy

Claims must reference files, tests, or governance artifacts instead of speculative wording.

Experience gate

  • Gate E1: The public guide, showcase, gallery pages, and case studies all link back to one another.
  • Gate E2: The layout remains readable on mobile and desktop.
  • Gate E3: The first visible content block explains why the page exists before deep links appear.

Safety gate

  • Gate S1: No secrets, credentials, or private runtime data appear on the public surface.
  • Gate S2: The page explicitly keeps display content separate from private operational notes.

What gets checked before publish

Navigation

Manual Home, Developer, User Guide, Public, Features, and GitHub are present and correctly linked.

Quality

Cards are meaningful, not placeholders; case studies and gallery pages contain inspectable material.
